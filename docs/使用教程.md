# NapCat File Mover 使用教程

这份教程面向本机运行的 Windows/macOS 用户。当前版本是 MVP：支持 NapCat 群文件扫描/搬运、白名单网页下载、本地归档、任务列表、重试/暂停/恢复、主题搜索和 GUI 配置。

## 1. 准备环境

需要先安装：

- Go
- Node.js 和 npm
- Wails CLI，用于桌面版打包
- NapCat，并启用 OneBot HTTP 能力

macOS 上可参考：

```bash
brew install go
go install github.com/wailsapp/wails/v2/cmd/wails@latest
```

如果 `wails` 命令找不到，把 Go 用户 bin 加到 shell PATH：

```bash
export PATH="$HOME/go/bin:$PATH"
```

## 2. Windows 快速开始

本节以 PowerShell 为例。建议使用普通用户权限运行；只有安装依赖或修改系统 PATH 时才可能需要管理员权限。

### 2.1 安装工具链

推荐用 `winget` 安装：

```powershell
winget install GoLang.Go
winget install OpenJS.NodeJS.LTS
winget install Git.Git
```

安装完成后，关闭并重新打开 PowerShell，验证：

```powershell
go version
node --version
npm --version
git --version
```

安装 Wails CLI：

```powershell
go install github.com/wailsapp/wails/v2/cmd/wails@latest
```

如果 `wails` 命令找不到，把 Go 用户 bin 加到当前用户 PATH。默认路径通常是：

```text
%USERPROFILE%\go\bin
```

可以在 PowerShell 中临时验证：

```powershell
$env:Path = "$env:USERPROFILE\go\bin;$env:Path"
wails version
```

要永久添加，打开：

```text
系统设置 -> 系统 -> 关于 -> 高级系统设置 -> 环境变量 -> 用户变量 Path
```

添加：

```text
%USERPROFILE%\go\bin
```

### 2.2 获取代码并进入目录

如果代码已经在本机，直接进入项目目录：

```powershell
cd "D:\path\to\资料搬运机器人"
```

路径里有中文或空格时，务必加英文双引号。

### 2.3 构建 GUI

```powershell
cd frontend
npm install
npm run build
cd ..
```

### 2.4 启动服务

```powershell
go run ./cmd/mover
```

看到类似输出表示启动成功：

```text
NapCatFileMover listening on http://127.0.0.1:8088
```

然后浏览器打开：

```text
http://127.0.0.1:8088
```

默认登录 token：

```text
change-me
```

### 2.5 Windows 配置文件位置

首次启动后会生成：

```text
%LOCALAPPDATA%\NapCatFileMover\config.yaml
```

在 PowerShell 中打开配置目录：

```powershell
explorer "$env:LOCALAPPDATA\NapCatFileMover"
```

常见文件：

```text
config.yaml   配置文件
mover.db      SQLite 数据库
cache\        下载缓存
files\        本地归档文件
logs\         日志目录
```

### 2.6 Windows 防火墙提示

本程序默认只监听：

```text
127.0.0.1:8088
```

这只允许本机访问。NapCat 和本程序在同一台 Windows 电脑上运行时，一般不需要开放公网防火墙。

如果你把监听地址改成 `0.0.0.0:8088` 或内网 IP，Windows 可能弹出防火墙提示。只有在你明确需要其他设备访问 GUI/API 时才允许。

### 2.7 Windows 下配置 NapCat

NapCat 的 OneBot HTTP 上报地址填写：

```text
http://127.0.0.1:8088/onebot/event
```

如果 NapCat 运行在同一台 Windows 机器上，`127.0.0.1` 就是正确地址。

如果 NapCat 在另一台机器，本程序就不能只监听 `127.0.0.1`，需要改成内网地址或 `0.0.0.0:8088`，并处理防火墙放行。这个模式不建议作为 MVP 默认部署。

## 3. 构建前端

首次运行前先生成 GUI 静态资源：

```bash
cd frontend
npm install
npm run build
cd ..
```

以后只要改了前端代码，就重新执行 `npm run build`。

## 4. 启动服务

开发/测试时运行：

```bash
go run ./cmd/mover
```

服务默认监听：

```text
http://127.0.0.1:8088
```

首次启动会自动生成配置文件：

- macOS：`~/Library/Application Support/NapCatFileMover/config.yaml`
- Windows：`%LOCALAPPDATA%/NapCatFileMover/config.yaml`

默认 GUI 登录 token 是：

```text
change-me
```

第一次登录后建议立刻在 GUI 的“设置”页改掉 `Admin Token`。

## 5. 配置 NapCat

在 NapCat 的 OneBot HTTP 配置中设置事件上报地址：

```text
http://127.0.0.1:8088/onebot/event
```

在本项目 GUI 的“设置”页填写：

- `NapCat 地址`：例如 `http://127.0.0.1:3000`
- `NapCat Token`：与 NapCat OneBot token 一致
- `管理员 QQ`：每行一个 QQ 号
- `允许群号`：每行一个授权群号
- `允许网站域名`：每行一个授权网站域名，例如 `example.com`

保存后重启服务，让 NapCat client、Worker 和限流配置完全生效。

## 6. GUI 使用

浏览器打开：

```text
http://127.0.0.1:8088
```

主要页面：

- `任务`：查看任务状态，执行重试、暂停、恢复。
- `搜索`：按主题搜索已扫描到的群文件，并批量创建搬运任务。
- `设置`：编辑 NapCat、管理员、白名单、Worker、搜索和存储参数。

配置页里的敏感项：

- `Admin Token` 留空表示不修改。
- `NapCat Token` 留空表示不修改。

## 7. 群命令

只有配置在 `管理员 QQ` 里的用户可以执行命令。

### 从白名单网站搬运到 QQ 群

```text
/搬运网页 https://example.com/file.zip 123456789
```

### 从白名单网站下载到本地归档

```text
/搬运网页 https://example.com/file.zip storage
```

### 扫描并搬运群文件

```text
/搬运群文件 <源群号> <目标群号|storage> [文件夹ID]
```

示例：

```text
/搬运群文件 111111111 storage
/搬运群文件 111111111 222222222
```

执行这个命令时，服务会同时把扫描到的群文件写入本地搜索索引。

### 搜索主题文件

```text
/搜索文件 高数期末复习资料
```

搜索会匹配：

- 文件名
- 目录名
- 中文规范化文本
- 拼音全拼
- 拼音首字母
- n-gram 模糊片段

### 按主题批量搬运

```text
/搬运主题 高数期末复习资料 storage
/搬运主题 高数期末复习资料 222222222
```

系统只会自动搬运高置信度结果。低置信度或没有命中的情况，会提示先用 `/搜索文件` 查看候选。

### 查询和重试任务

```text
/任务状态 1
/重试任务 1
```

## 8. 本地文件位置

默认目录：

- 配置：`config.yaml`
- 数据库：`mover.db`
- 下载缓存：`cache/`
- 本地归档：`files/`
- 日志目录：`logs/`

具体路径可在 GUI 设置页底部查看。

## 9. 打包桌面版

构建 macOS/Windows 桌面应用：

```bash
wails build
```

macOS 会生成 `.app`。Windows 打包需要在 Windows 环境执行，生成对应的 `.exe` 产物。

Windows 上建议在 PowerShell 中运行：

```powershell
wails build
```

构建完成后，产物通常在：

```text
build\bin\
```

如果 Wails 提示缺少 WebView2，安装 Microsoft Edge WebView2 Runtime。Windows 10/11 新版本通常已经内置。

当前 Wails 桌面壳会启动同一个本地 Go 服务，并绑定打开配置目录、打开日志目录、服务状态等动作。原生托盘菜单还没有接入稳定实现。

## 10. 常见问题

### 浏览器打不开 127.0.0.1:8088

先确认服务正在运行。可以测试：

```bash
curl http://127.0.0.1:8088/healthz
```

正常返回：

```json
{"ok":true}
```

### GUI 登录失败

检查 `config.yaml` 里的：

```yaml
app:
  admin_token: "..."
```

如果忘记 token，可以停止服务后手动修改配置文件，再重新启动。

### 群命令没有响应

依次检查：

- NapCat 是否在运行。
- OneBot HTTP 上报地址是否是 `/onebot/event`。
- 执行命令的 QQ 是否在管理员列表。
- 当前群是否在允许群号列表。
- NapCat token 是否与配置一致。

### `/搬运网页` 被拒绝

只允许下载白名单网站。把域名加入 GUI 设置页的“允许网站域名”，例如：

```text
example.com
files.example.org
```

不要填写完整 URL，只填域名。

### `/搜索文件` 搜不到

搜索索引来自群文件扫描。先执行：

```text
/搬运群文件 <源群号> storage
```

或后续接入专门的扫描命令后先扫描目录，再进行主题搜索。

### 修改设置后没有立即生效

当前 MVP 保存配置后需要重启服务。尤其是 NapCat 地址、token、Worker 并发、存储路径这类运行时组件配置。

### Windows 提示 `go`、`npm` 或 `wails` 不是内部或外部命令

关闭当前 PowerShell，重新打开后再试。

如果仍然失败，检查 PATH：

```powershell
$env:Path
```

通常需要包含：

```text
C:\Program Files\Go\bin
C:\Program Files\nodejs
%USERPROFILE%\go\bin
```

### Windows 路径里有中文导致命令失败

PowerShell 一般支持中文路径，但需要加双引号：

```powershell
cd "D:\资料搬运机器人"
```

如果仍遇到工具链兼容问题，可以把项目放到不含空格和特殊符号的路径，例如：

```text
D:\dev\napcat-file-mover
```

### Windows 端口 8088 被占用

查看占用：

```powershell
netstat -ano | findstr :8088
```

如果需要换端口，修改：

```text
%LOCALAPPDATA%\NapCatFileMover\config.yaml
```

例如：

```yaml
server:
  listen: "127.0.0.1:18088"
```

同时把 NapCat 上报地址改为：

```text
http://127.0.0.1:18088/onebot/event
```

### Windows Defender 报告未知应用

当前项目没有做代码签名，自己构建出来的 `.exe` 可能被 Windows 标记为未知发布者。确认来源是你本机源码构建后再允许运行。

## 11. 推荐使用流程

1. 启动服务。
2. 登录 GUI，把默认 token 改掉。
3. 填写 NapCat 地址/token。
4. 添加管理员 QQ、允许群号、允许网站域名。
5. 重启服务。
6. 在群里执行 `/搬运群文件 <源群号> storage` 建立索引和归档。
7. 用 `/搜索文件 <主题>` 验证主题搜索。
8. 用 `/搬运主题 <主题> <目标群号|storage>` 批量搬运。
