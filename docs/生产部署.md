# 生产部署

当前默认模式仍是 SQLite + 本地存储 + 数据库轮询，适合 Windows/macOS 单机桌面使用。生产模式可以启用 PostgreSQL、Redis Stream、S3/MinIO 和 Ollama 语义搜索。

## 启动依赖

```bash
docker compose -f docker-compose.production.yml up -d
```

首次使用 Ollama 语义搜索前拉取模型：

```bash
ollama pull bge-m3
```

如果 Ollama 跑在 Docker 容器里，也可以执行：

```bash
docker compose -f docker-compose.production.yml exec ollama ollama pull bge-m3
```

## 配置

复制 `configs/config.production.example.yaml` 到应用配置目录的 `config.yaml`，或者启动时指定配置文件：

```bash
go run ./cmd/mover -config configs/config.production.example.yaml
```

关键配置：

- `database.driver: postgres`
- `database.dsn: postgres://mover:mover@127.0.0.1:5432/mover?sslmode=disable`
- `redis.enabled: true`
- `storage.type: s3`
- `storage.s3.endpoint: http://127.0.0.1:9000`
- `search.semantic.enabled: true`
- `search.semantic.endpoint: http://127.0.0.1:11434`
- `search.semantic.model: bge-m3`

## 运行模式

数据库仍是任务状态真相源。Redis Stream 只负责多实例唤醒和派发，进程启动时会把数据库里 `pending` 的任务重新补发到 Stream，因此 Redis 丢消息不会导致任务丢失。

多实例 Worker 可以使用不同的 `redis.consumer_name`：

```yaml
redis:
  consumer_group: "mover-workers"
  consumer_name: "mover-2"
```

## 热重载

开启后程序会监听 `config.yaml`：

```yaml
reload:
  enabled: true
  debounce_ms: 500
```

可热重载：

- 管理员 QQ、群白名单、网站白名单
- 限流参数
- 搜索阈值和 Ollama 配置
- NapCat HTTP 客户端
- S3 客户端参数

需要重启：

- `server.listen`
- `database.driver/path/dsn`
- `redis.enabled/addr/db/stream/consumer_group`
- `storage.type`

也可以通过 GUI 或 HTTP 手动触发：

```bash
curl -X POST http://127.0.0.1:8088/api/config/reload
```

该接口需要登录 Cookie。

## 运行状态

GUI 使用：

```text
GET /api/runtime/status
```

返回当前数据库、队列、存储、语义搜索和热重载状态。语义搜索不可用时会自动降级为文件名、目录、拼音和模糊搜索。
