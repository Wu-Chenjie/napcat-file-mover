# 代码签名

仓库只提供签名脚本和流程说明，不包含任何证书、账号或密钥。未签名包可以用于本地测试；正式分发建议使用平台证书签名。

## macOS

需要 Apple Developer 账号、本机 Keychain 中的 Developer ID Application 证书，以及 `notarytool` 凭据配置。

环境变量：

```bash
export APPLE_SIGN_IDENTITY="Developer ID Application: Example Co (TEAMID)"
export APPLE_TEAM_ID="TEAMID"
export APPLE_NOTARY_PROFILE="notary-profile"
```

签名并公证：

```bash
./scripts/sign-macos.sh build/bin/NapCatFileMover.app
```

脚本会执行：

- `codesign --deep --options runtime --timestamp`
- `xcrun notarytool submit --wait`
- `xcrun stapler staple`
- `spctl --assess`

缺少环境变量时脚本会直接失败，不会生成半签名产物。

## Windows

需要 Windows SDK 的 `signtool.exe` 和 `.pfx` 代码签名证书。建议在 Developer PowerShell 中运行。

环境变量：

```powershell
$env:WINDOWS_CERT_PATH="C:\certs\codesign.pfx"
$env:WINDOWS_CERT_PASSWORD="change-me"
$env:WINDOWS_TIMESTAMP_URL="http://timestamp.digicert.com"
```

签名：

```powershell
.\scripts\sign-windows.ps1 -InputPath build\bin\NapCatFileMover.exe
```

脚本会使用 SHA256 摘要和时间戳服务器，并在签名后运行 `signtool verify /pa /v`。

## 自签名和未签名

自签名证书适合内部测试，但不会消除 Windows SmartScreen 或 macOS Gatekeeper 的外部分发提醒。未签名包也可以运行，但用户首次打开时可能需要手动允许。
